モバイルオーダーです。やんわりと実用化を試してきましたが今回のフレームワークが初めてベータ版公開までこぎつけた事案になります。
Postgresql10 32bitを使っていますがDelphi12の制約のためです。使ってみた感想postgresqlは堅牢な感じがします。軽くてバグが少ない。
正確に使えば意識せず力を添えてくれます。
ツールの一つであるmobileは厨房で利用するタブレット向けにコンパイルする予定です。
本当を言うとMac mobile orderよりも先に同様のアプリを発表したかったのですが、それがかなわなかったので一度しゃがんで力をためていこうと思います。
pwaには設定用のconfig.jsonファイルがついています。ここにあるテーブル番号によってお客様の識別をしています。
QRコードで座席設定できるように仕様が変更になりました。IPアドレスを収集するのでプライバシーポリシーの作成が必須です。

https://www.youtube.com/embed/hKLkVwcr3pg?si=MN-HSY_MBbCHzSgU

pwa=座席用のウェブアプリです。テーブル設置のタブレットを想定したデザインレイアウトですが使用するデバイスは個人のスマホでも可能なので
デザイン変更次第では使い方が変化します。ただしローカルネットワークに接続する必要があります。この点もサーバーのOSがサーバーライセンス
であるならば制約はありません。
mobile=厨房で注文を受ける用に開発されたアプリです。キッチンに備え付けのタブレットで運用します。ネーミングが紛らわしいですがサーバー内で
使うソフトウェアではないという感覚で命名しました。レジにあるサーバーPCでも起動させレジ機能を使います。全体の管理者として運用に携わる
人も本ソフトでチェックします。
menu=まず最初にテーブル識別用QRコードを生成させるための機能を持っています。データベースへのメニューの登録を行います。サーバーPC内で
動作します。営業開始前に仕込みの量を入力して在庫の管理を分担します。
cgi=サーバーPCにバックエンドとしてウェブサーバー登録します。このCGIでPostgreSQLにデータを入出力させます。
